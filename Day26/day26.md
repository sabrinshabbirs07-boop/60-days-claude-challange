# Prior Authorization Workflow Simulator

## Day 26 of 60 Days Claude Challenge

A single-file interactive healthcare workflow simulator that demonstrates the US Prior Authorization process through a gamified drag-and-drop experience.

This project simulates how patients, healthcare providers, and insurance payers interact during the Prior Authorization lifecycle. Users can manage different healthcare cases, complete documentation requirements, submit authorization requests, and experience different payer decisions.

The application is built completely using HTML, CSS, and Vanilla JavaScript without any external dependencies.

---

## Project Overview

Prior Authorization (PA) is a healthcare approval process where insurance companies review and approve certain medical services, procedures, or medications before they are provided.

This simulator transforms the complex PA workflow into an interactive educational experience where users can understand each stage of the authorization journey.

---

## Features

### Interactive Workflow Simulation

- Drag and drop patient cases through workflow stages
- Three healthcare workflow lanes:
  - Patient
  - Provider / Clinic
  - Payer / Insurance

### Patient Scenarios

The simulator includes multiple healthcare cases:

- Elective Surgery
- Diagnostic MRI
- Specialty Medication
- Inpatient Admission

Each scenario contains realistic medical information, documentation requirements, and review conditions.

---

## Prior Authorization Workflow

The application follows the complete authorization journey:

1. Patient Referral
2. Medical Necessity Evaluation
3. Document Collection
4. Prior Authorization Submission
5. Payer Clinical Review
6. Approval, Pending, or Denial Decision
7. Appeal and Peer-to-Peer Review Process

## Healthcare Simulation Features

The simulator supports multiple outcomes:
