Defend Your Experience

Product Requirements Document · v1.0 · AB Talks 60-Day Claude AI Challenge Capstone
One-line pitch: An AI interviewer that reads your resume and interrogates you on it — turning passive resume review into active interview defense practice for campus placements.

1. Problem Statement

Most resume tools help students write a better resume. None help students defend what's already on it. In real technical interviews, candidates are frequently caught off guard by follow-up questions like "walk me through how you actually built this" or "what was your individual contribution?" — because they've never been forced to justify their own claims out loud. Students over-embellish resumes with numbers and buzzwords they can't back up, and only discover this gap during a real, high-stakes interview.

2. Target Users

Primary

  • B.Tech students preparing for campus placements
  • Fresh graduates applying to entry-level tech roles
  • Early-career software engineers (0–2 yrs) prepping for interviews

Claims defended

  • Projects & GitHub work
  • Technical skills
  • Internships & certifications
  • Academic achievements
  • Problem-solving experience

3. Goals & Success Metrics (Day 10)

  • Deployed, publicly accessible web app (frontend + backend live)
  • User can paste a resume and complete a full 5–8 question adaptive chat interview
  • Defense Report generates reliably at session end (score dashboard + claim breakdown + coaching summary)
  • No critical bugs; graceful handling of AI API errors/rate limits
  • Session history persists across page reloads via local storage
  • UI is polished, responsive, and consistent with a premium dark aesthetic

4. Core User Flow (v1.0)

  1. User pastes resume/project/profile text into the app
  2. AI extracts 5–8 meaningful, defensible claims from the content
  3. AI interviewer opens the session and presents the first challenge based on a claim
  4. User responds conversationally in a chat interface
  5. AI silently scores the response (confidence, evidence, technical depth, clarity) and generates the next, deeper follow-up — adaptive to answer quality
  6. After 5–8 questions, session ends and the AI generates the Defense Report
  7. Report + session are saved to local storage; user can review past sessions or start fresh with a new profile

6. Explicitly Out of Scope (v1.0)

  • User accounts / authentication
  • Cloud sync / database persistence
  • Multiple simultaneous resume profiles
  • Company-specific interview modes
  • Voice-based interviews
  • Unlimited/open-ended session length
  • Multi-round interview simulations
  • Custom-built evaluation/ML engine (AI-generated weighted scoring only)

7. Functional Requirements

7.1 Input & Claim Extraction

  • System accepts pasted plain text (min/max length validated)
  • System sends resume text to AI API to extract a structured list of claims
  • Each claim includes: claim text, category (project/skill/internship/certification/academic/behavioral), and priority for interviewing

7.2 Adaptive Interview Engine

  • AI maintains conversation context across the full session
  • Each new question is generated based on: the current claim under discussion, the user's prior answer, and the overall session progress (question count)
  • Session hard-caps at 8 questions or ends early at 5 if coverage is sufficient
  • AI persona: skeptical-but-fair technical interviewer, not hostile

7.3 Scoring & Defense Report

  • Each answer scored on 4 axes: Confidence, Technical Depth, Communication, Evidence Strength (simple AI-generated weighted scoring, not a custom ML model)
  • Overall Interview Readiness % computed from aggregated axis scores
  • Report includes claim-by-claim analysis: convincing points, weak points, missing evidence, improvement suggestion
  • Report includes AI Coaching Summary: strongest areas, biggest risks, stories to prepare, concepts to revise

7.4 Persistence

  • Sessions and reports stored in browser local storage keyed by session ID/timestamp
  • User can view a list of past sessions and reopen any Defense Report
  • Replacing the active profile does not erase past session history

8. Non-Functional Requirements

  • Security: AI API keys never exposed client-side; all AI calls proxied through backend
  • Reliability: Graceful fallback messaging on API errors/timeouts/rate limits
  • Performance: Chat response latency communicated via visible "typing/thinking" indicator
  • Responsiveness: Usable on both desktop and mobile viewports
  • Accessibility: Keyboard-navigable chat input, sufficient color contrast on dark theme

9. High-Level Architecture

Frontend: HTML/CSS/JS SPA Backend: Node.js/Express or Python/Flask AI: Claude / OpenAI / Gemini API Storage: Browser localStorage Deployment: Vercel / Render

Frontend never calls the AI API directly. All prompts (claim extraction, interview turn generation, scoring, report generation) are constructed and sent server-side. Frontend communicates only with the backend's own endpoints.
